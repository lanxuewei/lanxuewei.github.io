一、整体介绍

首先简单介绍一下本文介绍的两个系统： Master 和 Slave ，Master为任务调度系统，主要负责对于任务进行分配、结果回收等操作；Slave为任务代理计算节点，主要负责任务的计算以及后续数据处理等。

二、模块概要

（1）任务源：其中任务源为产生任务的地方，每一个任务产生时便会将任务推送给 Master，其中推送的任务包含未计算与不可计算任务，未计算表示任务可计算但是未计算，所以该任务会经过 Master 分配给 Slave 进行计算；不可计算任务表示任务缺少关键内容，不可计算，无需分配给 Slave 计算，直接在Master生成失败一条失败任务即可。

（2）Master集群：Master 为任务分配调度中心，多台机器搭配 单个Mysql ，数据库主要记录信息为：

1、任务基本信息（业务主键、类型、状态、结果、失败原因、来源、分配机器计算节点Slave Ip 、计算尝试次数、任务分配尝试次数、创建以及分配完成各项时间等）

2、计算节点 Slave 基本信息（业务主键、计算节点 Ip 、上次以及下次心跳时间、Gpu 个数 [因为计算节点使用 ai 进行图片分析，所以 Gpu 个数作为任务分配时负载均衡的一个权衡因子]、运行任务个数、等待任务个数、繁忙程度、可处理任务类型）

3、消息队列信息（业务主键、消息主题、推送内容、附加信息、推送状态）

（3）Slave代理计算节点集群：Slave通过心跳方式向Master注册自己，报告自己当前状态，其中包括但不限于当前处理任务数、当前等待任务数、支持的任务类型以及Gpu个数等，当Master分配任务至Slave以后，Slave先进行启动Ai准备工作，准备就绪之后启动Ai进行计算，当数据库任务状态被更新后自动将结果或失败原因同步到Master中，同步完成则认为任务计算结束。

三、任务流转过程

首先任务通过任务源流入Master，在Master中的任务分为可启动任务和不可启动任务，本来按理说Master才是任务调度中心，任务能否启动应该由Master决定，但任务源在推送任务的时候已确定，原因在于，任务源在Master启动任务之前会进行许多的任务预先准备和过滤，如果缺失基本计算资源，比如文件缺失、文件受损等原因，那就直接在Master上创建失败任务即可，不需要再进行判断或计算，浪费资源，创建失败任务原因为所有的任务都可追溯。

其中Master和Slave都为集群部署，所以涉及到任务分配时并发问题，具体问题以及处理方式如下：

（1）Master分配任务过程如何实现负载均衡，如何分配任务更加合理，使得单个代理节点都资源最大化。

说明：Master进行任务分配时会进行负载均衡，其中权重因子计算方式是等待和处理任务数除以Gpu个数，即单个Gpu任务数，因为处理影像的Ai主要消耗Gpu资源；除此之外，后期数据积累后也可以考虑通过病种的类型，统计分析每个病种大概所需处理时间作为计算因子之一。

（2）Master分配任务是定时任务，定时查询需要计算任务和空闲代理计算资源，进行负载均衡后对任务进行分配处理，但是如果多个机器同时分配，就可能导致单个任务被多个Master分配到多个Slave中去，导致重复计算，浪费资源。

说明：Master中使用选举方式进行leader选举，挑选出leader执行定时任务，进行任务分配，非leader不执行该定时任务。

（3）Master分配任务至Slave时，每个Slave都有自己的数据库，但这样就导致任务回收时需要做废原任务，然后再分配给其他Slave，为何不直接所有Slave使用同一个数据库。

说明：一部分是历史原因，云平台的前身是单机版，单机版可以认为是类似于Slave，但自身包含了存储启动任务以及处理任务等逻辑，为了保持兼容性，将Slave与单机版保持接近便于迁移以及后期兼容；另一部分是为了减少数据库负载，分摊数据库压力，当单个代理节点挂掉时影响较小，其他代理节点还可以正常工作；

超时回收：目前任务有超时机制，当一个任务超时之后会进行重分配，并通知原任务做废，当Slave更新Ai结果回Master时，Master会通过任务所属Slave的Ip进行更新控制，保证Ai结果是最新分配的Slave所更新的。

灾难扩散：因为任务超时原因可能是任务计算失败，且有可能数据会导致代理计算节点挂掉，所以为了防止任务回收以后分给其他机器又导致其他机器挂掉，采用了尝试次数进行控制，防止灾难扩散问题，即单个任务最多只能分配三次，三次失败以后，该条任务不再分配处理。

五、改进

1、原Master交互方式，如下图：

优化方式：将Slave集群更新Ai结果以及发送心跳方式从Dobbo调用修改为Http接口调用，任务新增环境群组Id，假设目前环境为阿里云和华为云，且配置为：

aliyun.envGroupId=”http://cloud.ideaFly.com/master"
Huaweiyun.envGroupId=”http://cloud.ideaFly.com/master"

当Master分配任务时，将自己的任务群组Id带上，Slave收到任务进行计算，计算完毕将结果返回时，通过匹配任务对应的Master地址，将结果反馈给Master，其中环境群组Id为每个环境对外的开放接口，通过域名以及标记进行对应的转发。
